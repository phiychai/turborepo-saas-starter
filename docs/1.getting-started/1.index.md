# Quick Start Guide

Get your production-ready SaaS up and running in minutes!

## Prerequisites

- Node.js >= 18.0.0
- pnpm >= 9.0.0
- Docker & Docker Compose

## 1. Installation

### Quick Setup (Recommended)

Run our automated setup script:

```bash
# Clone the repository
git clone <your-repo-url>
cd turborepo-saas-starter

# Run setup script
./scripts/setup.sh
```

This script will:
- âœ… Copy all environment files
- âœ… Generate secure random secrets
- âœ… Install dependencies
- âœ… Set up permissions

### Manual Setup

If you prefer to set up manually:

```bash
# Clone the repository
git clone <your-repo-url>
cd turborepo-saas-starter

# Install dependencies
pnpm install

# Copy environment files
cp env.example .env
cp apps/backend/env.example apps/backend/.env
cp apps/web/env.example apps/web/.env

# Make scripts executable
chmod +x scripts/init-db.sh
```

## 2. Configure Environment

### Root Environment (Docker Compose)

Copy and edit the root `.env` file:

```bash
cp env.example .env
```

Edit `.env` and update the following (minimum required):

```bash
# Generate a secure APP_KEY
APP_KEY=$(node -e "console.log(require('crypto').randomBytes(32).toString('hex'))")

# Generate a JWT_SECRET
JWT_SECRET=$(node -e "console.log(require('crypto').randomBytes(32).toString('hex'))")

# Generate a DIRECTUS_SECRET
DIRECTUS_SECRET=$(node -e "console.log(require('crypto').randomBytes(32).toString('hex'))")

# Set strong passwords
POSTGRES_PASSWORD=your_secure_password
DIRECTUS_ADMIN_PASSWORD=your_secure_password
```

### Backend Environment (AdonisJS)

Copy and edit the backend `.env` file:

```bash
cp apps/backend/env.example apps/backend/.env
```

### Frontend Environment (Nuxt)

Copy and edit the frontend `.env` file:

```bash
cp apps/web/env.example apps/web/.env
```

## 3. Start Services

```bash
# Start all services in development mode
pnpm docker:up

# Wait for services to be ready (check logs)
pnpm docker:logs
```

## 5. Create Your First Admin User

You can create users via the API:

```bash
# Register a new user
curl -X POST http://localhost:3333/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "SecurePass123!",
    "fullName": "Admin User"
  }'
```

To make this user an admin, update the database:

```bash
# Access PostgreSQL
docker-compose exec postgres psql -U postgres -d adonis_db

# Update user role
UPDATE users SET role = 'admin' WHERE email = 'admin@example.com';

# Exit
\q
```

## Troubleshooting

### Services won't start

```bash
# Check if ports are already in use
lsof -i :3000  # Nuxt
lsof -i :3333  # AdonisJS
lsof -i :5432  # PostgreSQL
lsof -i :8055  # Directus

# Stop conflicting services or change ports in .env
```

### Database connection errors

```bash
# Check PostgreSQL is running
docker-compose ps postgres

# View PostgreSQL logs
docker-compose logs postgres

# Restart PostgreSQL
docker-compose restart postgres
```

### Authentication not working

1. Check cookies are being set (browser dev tools)
2. Verify APP_KEY is set in .env
3. Check Redis is running: `docker-compose ps redis`
4. Verify session driver is set to 'redis'

### Can't access Directus

1. Verify Directus is running: `docker-compose ps directus`
2. Check Directus logs: `docker-compose logs directus`
3. Wait for database initialization (first start takes longer)
4. Verify PostgreSQL database 'directus_db' exists

## Key Endpoints Reference

### Authentication
```
POST   /api/auth/register         - Create account
POST   /api/auth/login            - Login
POST   /api/auth/logout           - Logout
GET    /api/auth/me               - Current user
POST   /api/auth/change-password  - Change password
```

### User Management
```
GET    /api/user/profile          - Get profile
PATCH  /api/user/profile          - Update profile
GET    /api/user/users            - List users (admin)
PATCH  /api/user/users/:id/...    - Manage users (admin)
```

### CMS (Directus Proxy)
```
GET    /api/cms/collections/:name      - Get collection
GET    /api/cms/collections/:name/:id  - Get item
POST   /api/cms/:collection            - Create item
PATCH  /api/cms/:collection/:id        - Update item
DELETE /api/cms/:collection/:id        - Delete item
```

### Billing (Lago Integration)
```
GET    /api/billing/account           - Get/create customer in Lago
GET    /api/billing/subscriptions     - Get customer subscriptions
POST   /api/billing/subscriptions     - Create subscription
DELETE /api/billing/subscriptions/:id - Cancel subscription
GET    /api/billing/invoices          - Get customer invoices
GET    /api/billing/plans             - Get available billing plans
POST   /api/billing/usage             - Track usage events (metered billing)
GET    /api/billing/payment-provider  - Get payment provider status
```
